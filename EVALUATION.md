## 二、功能性评估 ★★★★☆

### ✅ 已实现且完善
- **讲座信息流**：按日期分组、倒序排列、sticky 日期条、状态标签（即将开始/已结束）
- **多维筛选**：校区（5 个）、学院（点击 Tag 直达）、年份下拉、关键词搜索（标题/主讲人/题目等多字段）
- **统计页**：学院 × 年份交叉矩阵，支持按学院名/总数排序，底部合计行
- **本地点赞**：localStorage 去重，同一浏览器不可重复点赞
- **增量爬虫**：URL 去重 + 时间基线（last_scrape.json），跳过已抓条目，OCR 仅对海报图触发
- **每日自动更新**：GitHub Actions 定时（北京 03:00）+ 手动触发
- **OCR 兜底**：正文不足 50 字时对海报图片做 RapidOCR

### ⚠️ 功能局限（非缺陷，属架构取舍）
1. **点赞不跨设备共享**：当前点赞存浏览器 localStorage，公网访客看到的"点赞数"实为该浏览器本地计数（初始为 0）。若要真实社交点赞需后端存储。
2. **公网"立即抓取"需额外配置**：静态托管下「抓取新数据」按钮默认提示"已配置每日自动更新"；要实现"点一下立即更新"需部署一个持有 PAT 的代理（已预留 `WORKFLOW_DISPATCH_URL` 接口）。
3. **官微讲座未接入**：微信公众号无公开 API，自动抓取受限（已存档官微清单，后续可按需接入）。
4. **lectureEnd 填充率 32%**：部分来源页未标注结束时间，属数据源限制。

### 评分依据
核心功能完整、筛选交互流畅、数据更新自动化已就位。扣分项为点赞的本地化局限和官微覆盖缺口。

## 三、安全性评估 ★★★★☆

### ✅ 已加固（详见 SECURITY.md）
| 风险 | 处置 |
|------|------|
| 信息源管理入口暴露 | 从公网页面移除 + .gitignore 排除 sources.html，永不进公网仓库 |
| 前端 XSS（链接协议） | `safeUrl()` 仅放行 http/https，阻断 `javascript:`/`data:` |
| HTML 注入 | 全站无 `v-html`，所有外部数据经 Vue `{{}}` 自动转义；实测 115 条标题零 HTML 标签 |
| 内容安全策略 | index.html / stats.html 均配 CSP meta + `referrer: no-referrer` |
| 后端绑定范围 | server.py 默认绑 127.0.0.1，防局域网暴露写操作后台 |
| 敏感文件泄露 | .gitignore 排除 .workbuddy/、wechat_accounts.md、.env、*token*.txt 等 |
| CI 工作流权限 | 仅用 GITHUB_TOKEN（contents:write 最小权限），无需任何用户密钥 |
| 并发写冲突 | workflow concurrency 保证同一时刻只跑一个抓取 |

### ⚠️ 残留风险（可接受/需部署时注意）
1. **CSP 含 `unsafe-eval`**：Tailwind Play CDN 和 Vue 运行时编译需要。这是 CDN 免构建方案的固有代价。若要消除，需改用本地构建（Tailwind CLI + Vue 预编译），工程量中等。
2. **静态托管无法设 HTTP 安全头**：GitHub Pages 不支持自定义 `X-Frame-Options`/`HSTS`/`X-Content-Type-Options`。点击劫持风险低（站点无敏感操作），如在意可前置 Cloudflare 免费层注入安全头。
3. **Token 暴露风险（仅启用"立即更新"时）**：若未来配置 `WORKFLOW_DISPATCH_URL`，PAT 绝不能写进前端 JS（已注释警示），须用无服务器代理持有 + 频率限制。

### 评分依据
公网是纯静态只读站，无后端写攻击面；XSS/注入/CSP/敏感文件均已处理。扣分项为 unsafe-eval 和无自定义响应头，均有明确缓解路径。

## 四、可用性评估 ★★★★☆

### ✅ 表现良好
- **响应式适配**：桌面/移动端校区导航分离（桌面横排、移动端横滑）；内容区自适应
- **视觉一致性**：华师蓝主调、校徽+书法标题、校训 fixed 悬浮（宽屏）、扁平信息流
- **筛选体验**：多维筛选即时响应、清除按钮、筛选计数显示
- **加载容错**：API 不可用时自动回退读静态 lectures.json，静态部署也能正常显示
- **无构建依赖**：CDN + 原生 JS，部署极简、无构建失败风险

### ⚠️ 可改进
1. **下拉菜单仅 hover 触发**：键盘用户和触屏设备体验略差（已有 click 切换兜底，但 focus 管理不完善）。
2. **无分页/懒加载**：当前 115 条全量渲染，性能无问题；若数据增长到 500+ 条可能需虚拟滚动。
3. **无 404 页面**：GitHub Pages 用默认 404，可加 `site/404.html` 提升体验（可选）。
4. **无 SEO/社交元信息**：缺 og:title / description，分享到社交平台无预览卡片（讲座聚合站影响小）。

### 评分依据
核心交互流畅、视觉统一、容错完善。扣分项为可访问性细节和未来扩展性预留。
